# 第 1 步开发计划定稿：.NET 10 解决方案骨架与可观测性基础

> **版本**：V3.0  
> **对应总架构**：`1、架构最终确定版.md` V3.0（2026 .NET 10 生产化修正版）  
> **本步目标**：创建 `IoTHunter` 解决方案、公共类库、三个服务项目，完成 .NET 10 基线、共享事件模型、Serilog 日志关联、OpenTelemetry 指标/追踪、Prometheus `/metrics` 端点和 DeviceSimulator 通用主机防退出骨架。  
> **本步边界**：本步只铺设工程骨架和可观测性基础，不实现 Kafka 生产、MQTT 接入、PG/Redis 写入、设备认证和 K8s 部署。

---

## 0. 审核结论与必要调整

原第一步计划不能直接作为 V3.0 总架构的编码依据，主要问题如下：

1. **运行时不一致**：原计划仍使用 .NET 8 时代包版本和 `Microsoft.Extensions.* 8.0.0`，与总架构 `.NET 10 LTS` 基线不一致。
2. **依赖版本风险**：原计划手写固定老版本，如 `OpenTelemetry 1.8.1`、`Confluent.Kafka 2.3.0`、`Npgsql 8.0.3`。第一步应使用 NuGet 当前稳定版本，避免人为锁死在旧生态。
3. **存储依赖过早**：总架构已改为优先 `PostgreSQL/TimescaleDB`，`InfluxDB/ClickHouse` 只是可选时序投影。第一步不应强制安装 `InfluxDB.Client`。
4. **本地端口冲突**：原计划让 `IoTGateway` 和 `BackendProcessor` 都监听 `localhost:80`，本地无法同时启动。第一步改为本地开发端口 `5080` 和 `5081`，容器/K8s 阶段再统一映射到 80/9464。
5. **缺少共享事件模型**：总架构要求 `event_id`、`sequence`、`recorded_at`、`schema_version` 等字段。第一步应先放入 `IoTHunter.Shared`，后续 Kafka、PG、Redis、Simulator 都复用。
6. **DeviceSimulator 闪退问题判断正确**：通用主机必须注册 `IHostedService`。本定稿保留该修正，但改为独立 `PlaceholderWorker.cs`，避免把过多类型塞进 `Program.cs`。

## 1.6 编写共享领域模型

删除 `IoTHunter.Shared/Class1.cs`。

新建 `IoTHunter.Shared/Domain/ReliabilityLevel.cs`：

```csharp
namespace IoTHunter.Shared.Domain;

public enum ReliabilityLevel
{
    BestEffort = 0,
    AtLeastOnce = 1,
    Critical = 2
}
```

新建 `IoTHunter.Shared/Domain/TelemetryEnvelope.cs`：

```csharp
namespace IoTHunter.Shared.Domain;

public sealed record TelemetryEnvelope
{
    public required string EventId { get; init; }

    public required string DeviceId { get; init; }

    public long Sequence { get; init; }

    public required string MetricType { get; init; }

    public required DateTimeOffset RecordedAt { get; init; }

    public required string PayloadJson { get; init; }

    public DateTimeOffset ReceivedAt { get; init; }

    public required ReliabilityLevel ReliabilityLevel { get; init; }

    public int SchemaVersion { get; init; } = 1;

    public static TelemetryEnvelope Create(
        string eventId,
        string deviceId,
        string metricType,
        DateTimeOffset recordedAt,
        string payloadJson,
        ReliabilityLevel reliabilityLevel,
        long sequence = 0)
    {
        return new TelemetryEnvelope
        {
            EventId = eventId,
            DeviceId = deviceId,
            MetricType = metricType,
            RecordedAt = recordedAt,
            PayloadJson = payloadJson,
            ReliabilityLevel = reliabilityLevel,
            Sequence = sequence,
            ReceivedAt = DateTimeOffset.UtcNow,
        };
    }
}
```

设计要求：

- `EventId` 是后续 Kafka、PG、Redis、DLQ 重放的幂等核心。
- `RecordedAt` 来自设备采集时间，`ReceivedAt` 由静态工厂方法 `Create()` 自动设置为 `DateTimeOffset.UtcNow`，**不使用属性默认值**以避免 C# record 的 `with` 表达式重新求值默认值导致时间戳被意外覆盖。
- `ReliabilityLevel` 为 `required`，**禁止默认值**，强制调用方显式声明可靠性级别，防止关键事件被静默降级为 `BestEffort`（对齐 ADR-014）。
- `PayloadJson` 第一阶段先保持灵活，后续可以演进为强类型 payload 或 schema registry。
- 推荐使用 `TelemetryEnvelope.Create(...)` 工厂方法构建实例，避免遗漏 `ReceivedAt` 或 `ReliabilityLevel` 的设置。

## 1.7 编写共享日志与可观测性基础设施

新建 `IoTHunter.Shared/Infrastructure/TraceIdEnricher.cs`：

```csharp
using Serilog.Core;
using Serilog.Events;
using System.Diagnostics;

namespace IoTHunter.Shared.Infrastructure;

public sealed class TraceIdEnricher : ILogEventEnricher
{
    public void Enrich(LogEvent logEvent, ILogEventPropertyFactory propertyFactory)
    {
        var activity = Activity.Current;

        if (activity is null)
        {
            return;
        }

        logEvent.AddPropertyIfAbsent(propertyFactory.CreateProperty("TraceId", activity.TraceId.ToString()));
        logEvent.AddPropertyIfAbsent(propertyFactory.CreateProperty("SpanId", activity.SpanId.ToString()));
    }
}
```

新建 `IoTHunter.Shared/Infrastructure/LoggingDefaults.cs`：

```csharp
using Serilog;

namespace IoTHunter.Shared.Infrastructure;

public static class LoggingDefaults
{
    public const string ConsoleTemplate =
        "[{Timestamp:HH:mm:ss} {Level:u3}] trace={TraceId} span={SpanId} {Message:lj}{NewLine}{Exception}";

    public static LoggerConfiguration ConfigureBaseLogger(LoggerConfiguration configuration)
    {
        return configuration
            .Enrich.With(new TraceIdEnricher())
            .WriteTo.Console(outputTemplate: ConsoleTemplate);
    }
}
```

新建 `IoTHunter.Shared/Infrastructure/OpenTelemetryDefaults.cs`：

```csharp
using Microsoft.Extensions.Configuration;
using Microsoft.Extensions.DependencyInjection;
using OpenTelemetry;
using OpenTelemetry.Metrics;
using OpenTelemetry.Resources;
using OpenTelemetry.Trace;

namespace IoTHunter.Shared.Infrastructure;

public static class OpenTelemetryDefaults
{
    public static OpenTelemetryBuilder AddIoTHunterOpenTelemetry(
        this IServiceCollection services,
        IConfiguration configuration,
        string serviceName)
    {
        var otlpEndpoint = configuration["OpenTelemetry:OtlpEndpoint"];

        return services.AddOpenTelemetry()
            .ConfigureResource(resource => resource.AddService(serviceName))
            .WithTracing(tracing =>
            {
                tracing
                    .AddHttpClientInstrumentation()
                    .AddSource(serviceName);

                if (!string.IsNullOrWhiteSpace(otlpEndpoint))
                {
                    tracing.AddOtlpExporter(options => options.Endpoint = new Uri(otlpEndpoint));
                }
            })
            .WithMetrics(metrics =>
            {
                metrics
                    .AddHttpClientInstrumentation()
                    .AddRuntimeInstrumentation();
            });
    }
}
```

注意：

- Web 项目还需要在各自 `Program.cs` 中继续链式添加 `AddAspNetCoreInstrumentation()` 和 `AddPrometheusExporter()`。
- `OpenTelemetryDefaults` 放在共享类库中，因此 `IoTHunter.Shared` 必须安装 Serilog、OpenTelemetry、HTTP/Runtime instrumentation 和相关 `Microsoft.Extensions.*.Abstractions` 包。

var builder = WebApplication.CreateBuilder(args);

    LoggingDefaults.ConfigureBaseLogger(configuration);

    .AddIoTHunterOpenTelemetry(builder.Configuration, "IoTGateway")
    .WithTracing(tracing => tracing.AddAspNetCoreInstrumentation())

新建 `DeviceSimulator/PlaceholderWorker.cs`：

```csharp
using Microsoft.Extensions.Hosting;
using Microsoft.Extensions.Logging;

namespace DeviceSimulator;

public sealed class PlaceholderWorker(ILogger<PlaceholderWorker> logger) : BackgroundService
{
    protected override async Task ExecuteAsync(CancellationToken stoppingToken)
    {
        logger.LogInformation("DeviceSimulator placeholder worker started.");

        try
        {
            await Task.Delay(Timeout.InfiniteTimeSpan, stoppingToken);
        }
        catch (OperationCanceledException) when (stoppingToken.IsCancellationRequested)
        {
            logger.LogInformation("DeviceSimulator placeholder worker stopped.");
        }
    }
}
```

  }
}

## 主要审核结论

原第一步计划不能直接沿用，主要问题是仍停留在旧版技术基线：使用 .NET 8 时代依赖版本、强制安装 `InfluxDB.Client`、两个 Web 服务都监听 `localhost:80`、缺少 V3.0 架构要求的共享事件模型，并且可观测性代码放在共享类库但未给共享类库安装对应依赖。

其中 DeviceSimulator “通用主机会闪退”的判断是正确的，我保留了这个修正，但改成独立 `PlaceholderWorker.cs`，更适合后续扩展成真实压测 Worker。
