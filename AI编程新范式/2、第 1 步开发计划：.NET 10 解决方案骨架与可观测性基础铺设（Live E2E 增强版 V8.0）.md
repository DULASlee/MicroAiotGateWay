# 第 1 步开发计划：.NET 10 解决方案骨架与可观测性基础铺设（Live E2E 增强版 V8.0）

> **版本**：V8.0（熔接两次裁决：Shared/Infrastructure分离、OTel全链路注册、TimescaleDB预留、编译严格模式）  
> **对应总架构**：`架构设计文档 V7.0` 第 1 步  
> **前置依赖**：第 0 步全部通过（所有中间件可达）  
> **本步目标**：创建 `IoTHunter` 解决方案，包含 **Shared（纯领域模型）** 和 **Infrastructure（可观测性基础设施）** 两个类库，以及三个服务项目。完成 .NET 10 严格模式编译基线、Serilog 日志关联（TraceId/SpanId）、OpenTelemetry 指标/追踪（含自定义 ActivitySource 注册）、Prometheus `/metrics` 端点，并完成与 Kafka 的冒烟连接。  
> **本步边界**：只铺设工程骨架和可观测性基础，不实现 Kafka 生产/消费、MQTT 接入、数据库写入、设备认证和 K8s 部署。  
> **核心禁令**：  
> - **硬编码禁令（ADR-022）**：本步所有连接串默认值必须通过 `IConfiguration` 注入，禁止在业务代码中硬编码。  
> - **Mock 数据禁令（ADR-023）**：本步验证必须使用真实的 Kafka、Mosquitto、PostgreSQL、Redis 中间件，禁止使用内存集合或假数据替代。  
> **【V8.0 关键变更】**：  
> - **Shared/Infrastructure 分离（ADR-027）**：`IoTHunter.Shared` 只保留纯 C# 领域模型（`TelemetryEnvelope`、`ReliabilityLevel`、`ReliabilityConfiguration`、`ConfigurationConstants`）。日志和 OTel 相关代码移入新建的 `IoTHunter.Infrastructure` 类库。  
> - **OTel 自定义 ActivitySource 注册（ADR-004 强化）**：在 `OpenTelemetryDefaults` 中显式注册服务级、Kafka Producer、Persistence Worker、Projection Worker 的 ActivitySource 名称，杜绝 Span 断链。  
> - **编译严格模式**：`Directory.Build.props` 增加 `<AnalysisMode>All</AnalysisMode>`，启用 .NET 10 最严格代码分析。  
> - **TimescaleDB 预留**：`docker-compose.local.yml` 增加 `timescaledb` 服务定义（端口 5433），`ConfigurationConstants` 增加对应连接串。

namespace IoTHunter.Shared.Domain;

public sealed record TelemetryEnvelope
{
    public required string EventId { get; init; }
    public required string DeviceId { get; init; }
    public long Sequence { get; init; }
    public required string MetricType { get; init; }
    public required DateTimeOffset RecordedAt { get; init; }
    public DateTimeOffset ReceivedAt { get; init; } = DateTimeOffset.UtcNow;
    public ReliabilityLevel ReliabilityLevel { get; init; } = ReliabilityLevel.BestEffort;
    public int SchemaVersion { get; init; } = 1;
    public required string PayloadJson { get; init; }

    /// <summary>
    /// 工厂方法：自动设置 ReceivedAt 为当前 UTC 时间。
    /// 禁止使用 new TelemetryEnvelope{} 绕过此方法。
    /// </summary>
    public static TelemetryEnvelope Create(
        string eventId,
        string deviceId,
        string metricType,
        DateTimeOffset recordedAt,
        string payloadJson,
        ReliabilityLevel reliabilityLevel = ReliabilityLevel.BestEffort,
        long sequence = 0)
    {
        return new TelemetryEnvelope
        {
            EventId = eventId,
            DeviceId = deviceId,
            Sequence = sequence,
            MetricType = metricType,
            RecordedAt = recordedAt,
            ReceivedAt = DateTimeOffset.UtcNow,
            PayloadJson = payloadJson,
            ReliabilityLevel = reliabilityLevel,
            SchemaVersion = 1
        };
    }
}
```

using Serilog.Core;
using Serilog.Events;
using System.Diagnostics;

namespace IoTHunter.Infrastructure;

public sealed class TraceIdEnricher : ILogEventEnricher
{
    public void Enrich(LogEvent logEvent, ILogEventPropertyFactory propertyFactory)
    {
        var activity = Activity.Current;

        if (activity is null)
        {
            return;
        }

        logEvent.AddPropertyIfAbsent(propertyFactory.CreateProperty("TraceId", activity.TraceId.ToString()));
        logEvent.AddPropertyIfAbsent(propertyFactory.CreateProperty("SpanId", activity.SpanId.ToString()));
    }
}
```

#### 1.5.3 OpenTelemetryDefaults（V8.0 强化：注册所有自定义 ActivitySource）

新建 `IoTHunter.Infrastructure/OpenTelemetryDefaults.cs`，粘贴以下内容：

```csharp
using Microsoft.Extensions.Configuration;
using Microsoft.Extensions.DependencyInjection;
using OpenTelemetry;
using OpenTelemetry.Metrics;
using OpenTelemetry.Resources;
using OpenTelemetry.Trace;

namespace IoTHunter.Infrastructure;

public static class OpenTelemetryDefaults
{
    public static OpenTelemetryBuilder AddIoTHunterOpenTelemetry(
        this IServiceCollection services,
        IConfiguration configuration,
        string serviceName)
    {
        var otlpEndpoint = configuration["OpenTelemetry:OtlpEndpoint"];

        return services.AddOpenTelemetry()
            .ConfigureResource(resource => resource.AddService(serviceName))
            .WithTracing(tracing =>
            {
                tracing
                    .AddHttpClientInstrumentation()
                    .AddSource(serviceName)                     // 服务级别
                    .AddSource($"{serviceName}.Kafka")         // Kafka Producer（第2步）
                    .AddSource($"{serviceName}.Persistence")   // 持久化 Worker（第3步）
                    .AddSource($"{serviceName}.Projection");   // 投影 Worker（第3步）

                if (!string.IsNullOrWhiteSpace(otlpEndpoint))
                {
                    tracing.AddOtlpExporter(options => options.Endpoint = new Uri(otlpEndpoint));
                }
            })
            .WithMetrics(metrics =>
            {
                metrics
                    .AddHttpClientInstrumentation()
                    .AddRuntimeInstrumentation();
            });
    }
}
```

using Microsoft.Extensions.Hosting;
using Microsoft.Extensions.Logging;

namespace DeviceSimulator;

public sealed class PlaceholderWorker(ILogger<PlaceholderWorker> logger) : BackgroundService
{
    protected override async Task ExecuteAsync(CancellationToken stoppingToken)
    {
        logger.LogInformation("DeviceSimulator placeholder worker started.");

        try
        {
            await Task.Delay(Timeout.InfiniteTimeSpan, stoppingToken);
        }
        catch (OperationCanceledException) when (stoppingToken.IsCancellationRequested)
        {
            logger.LogInformation("DeviceSimulator placeholder worker stopped.");
        }
    }
}
