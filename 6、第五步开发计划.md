## 5.1 子步骤一：IoTGateway Dockerfile

**目标**：为 IoTGateway 编写多阶段 Dockerfile，使用 .NET 10 SDK 构建、ASP.NET Core 10 runtime 运行，常规多文件发布，暴露 80 和 9464 端口。

### 5.1.1 动作项

1. **新建 `IoTGateway/Dockerfile`**

   ```dockerfile
   FROM mcr.microsoft.com/dotnet/sdk:10.0 AS build
   WORKDIR /src
   
   COPY IoTHunter.Shared/ IoTHunter.Shared/
   COPY IoTGateway/ IoTGateway/
   
   WORKDIR /src/IoTGateway
   RUN dotnet restore IoTGateway.csproj
   RUN dotnet publish IoTGateway.csproj -c Release -o /app/publish --no-restore
   
   FROM mcr.microsoft.com/dotnet/aspnet:10.0 AS runtime
   WORKDIR /app
   EXPOSE 80
   EXPOSE 9464
   
   ENV DOTNET_SYSTEM_GLOBALIZATION_INVARIANT=1
   ENV DOTNET_EnableDiagnostics=0
   ENV ASPNETCORE_URLS=http://+:80
   ENV OpenTelemetry__OtlpEndpoint=http://otel-collector:4317
   
   COPY --from=build /app/publish .
   ENTRYPOINT ["dotnet", "IoTGateway.dll"]
   ```

   **关键设计决策**：
   - `DOTNET_SYSTEM_GLOBALIZATION_INVARIANT=1`：剔除 ICU，节省 ~30MB。
   - `DOTNET_EnableDiagnostics=0`：关闭托管诊断，减少 CPU 开销。
   - OTLP Endpoint 默认指向 K8s Service `otel-collector:4317`，本地调试可覆盖为空。

### 5.1.2 验证标准

- `docker build -t iot-gateway:latest -f IoTGateway/Dockerfile .` 构建成功。
- `docker run -d -p 5080:80 -p 9464:9464 --name iot-gateway-test iot-gateway:latest` 启动成功。
- `curl http://localhost:5080/health/ready` 返回 Healthy。
- `curl http://localhost:9464/metrics` 输出 Prometheus 格式指标。

## 5.4 子步骤四：Mosquitto 容器化（已集成的清言修正，关键）

**目标**：基于 `eclipse-mosquitto:2.0.18` 镜像，注入完整配置、ACL，密码在 Docker 构建时直接哈希生成。

### 5.4.1 动作项

1. **新建 `infra/mosquitto/` 目录**

2. **编写 `infra/mosquitto/mosquitto.conf`**

   ```text
   persistence true
   persistence_location /mosquitto/data/

   listener 1883
   protocol mqtt

   listener 8083
   protocol websockets

   allow_anonymous false
   password_file /mosquitto/config/passwordfile
   acl_file /mosquitto/config/aclfile
   ```

3. **编写 ACL 文件 `infra/mosquitto/aclfile`**

   ```text
   user iotgateway
   topic read $share/gateway-group/device/+/telemetry
   topic read $share/gateway-group/device/+/event/critical

   user devicesim
   topic write device/+/telemetry
   topic write device/+/event/critical
   ```

4. **编写 `infra/mosquitto/Dockerfile`（已修正）**

   ```dockerfile
   FROM eclipse-mosquitto:2.0.18
   
   COPY mosquitto.conf /mosquitto/config/mosquitto.conf
   
   # 修正：使用 -b 模式逐行生成哈希密码，不再依赖外部明文 passwordfile
   RUN touch /mosquitto/config/passwordfile && \
       mosquitto_passwd -b -c /mosquitto/config/passwordfile iotgateway changeme && \
       mosquitto_passwd -b /mosquitto/config/passwordfile devicesim changeme
   
   COPY aclfile /mosquitto/config/aclfile
   
   RUN chmod 644 /mosquitto/config/mosquitto.conf /mosquitto/config/passwordfile /mosquitto/config/aclfile
   
   EXPOSE 1883
   EXPOSE 8083
   ```

   **注意**：`infra/mosquitto/passwordfile` 明文文件不再需要，直接删除。

### 5.4.2 验证标准

- `docker build -t mosquitto-iot:latest -f infra/mosquitto/Dockerfile infra/mosquitto/` 成功。
- `docker run -d -p 1883:1883 -p 8083:8083 --name mos-test mosquitto-iot:latest` 启动。
- 匿名发布被拒绝；`-u devicesim -P changeme` 发布成功。
- WebSocket 8083 端口可连接。

## 5.7 子步骤七：镜像导入 K3s containerd

**目标**：导出 tar 并导入 K3s 节点，不依赖外部 Registry。

### 5.7.1 动作项

1. **新建 `scripts/import-to-k3s.ps1`**

   ```powershell
   param(
       [string]$Tag = "latest"
   )
   
   $images = @("iot-gateway", "backend-processor", "device-simulator", "mosquitto-iot")
   $tars = @()
   
   foreach ($img in $images) {
       $tar = "$img-$Tag.tar"
       docker save -o $tar "$img`:$Tag"
       $tars += $tar
   }
   
   Write-Host "Copying images to K3s node..."
   scp $tars k3s-node:/tmp/
   
   Write-Host "Importing images into containerd..."
   ssh k3s-node "sudo ctr -n k8s.io images import /tmp/*.tar"
   
   Write-Host "Cleaning up..."
   Remove-Item $tars
   ssh k3s-node "rm /tmp/*.tar"
   ```

### 5.7.2 验证标准

- 脚本执行无报错。
- K3s 节点执行 `crictl images | grep -E "iot-gateway|backend-processor|device-simulator|mosquitto-iot"`，四个镜像均可见。
